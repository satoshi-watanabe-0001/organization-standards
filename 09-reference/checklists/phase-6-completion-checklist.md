# Phase 6 完了チェックリスト

> 運用監視と保守体制の確立完了確認

**目的**: Phase 6の運用体制が整い、PBIが完全にクローズできる状態であることを確認  
**使用タイミング**: PBI完了・クローズ前  
**参照元**: [phase-6-operations-maintenance-guide.md](../../00-guides/phase-guides/phase-6-operations-maintenance-guide.md)

---

## ✅ 使い方

このチェックリストは、Phase 6（運用・保守）のすべてのタスクが完了し、次のフェーズに移行する前に使用します。

### チェック方法
1. 各項目を順番に確認
2. 完了している項目には `[x]` をマーク
3. すべての項目が完了したら、次のフェーズへ進行可能

### 未完了項目がある場合
- 該当項目の担当者に確認
- 必要に応じてタスクを追加・完了
- すべて完了するまで次フェーズへ進まない

## 🔄 Phase 6 のワークフロー: 6ステップ

### **Step 6.1: モニタリング設定（初期設定）** (30-45分)

- [ ] 主要メトリクスを設定（CPU、メモリ、ディスク、ネットワーク）
- [ ] アプリケーションメトリクスを設定（レスポンスタイム、エラー率等）
- [ ] アラート閾値を設定
- [ ] アラート通知先を設定（Slack、メール等）
- [ ] ダッシュボードを構築
- [ ] ヘルスチェックを設定
### **Step 6.2: ログ管理設定（初期設定）** (20-30分)

- [ ] アプリケーションログを集中管理システムに送信
- [ ] エラーログのフィルタリングルールを設定
- [ ] ログレベル（DEBUG、INFO、WARN、ERROR）を適切に設定
- [ ] ログ保持期間を設定
- [ ] ログ分析ダッシュボードを構築
### **Step 6.3: 日常モニタリング（継続的）** (15-30分/日)

- [ ] 主要メトリクスをダッシュボードで確認
- [ ] アラートが発生していないか確認
- [ ] エラーログを確認
- [ ] パフォーマンス低下がないか確認
- [ ] 異常なトレンドがないか確認
### **Step 6.4: インシデント対応（発生時）** (可変: 15分-数時間)

- [ ] 初動対応と影響範囲の特定
- [ ] 事後分析（ポストモーテム）
- [ ] インシデントの重大度を評価（Critical / High / Medium / Low）
- [ ] 影響範囲を特定（ユーザー数、機能等）
- [ ] 初動対応を実施（ロールバック、サービス停止等）
- [ ] 根本原因を調査
- [ ] 恒久対策を実施
- [ ] インシデントレポートを作成
- [ ] ポストモーテムを実施
- [ ] 検知 → 2. 評価 → 3. 初動対応 → 4. 復旧 → 5. 事後分析 → 6. 再発防止

## 再発防止策

### **Step 6.5: 変更管理（必要時）** (1-2時間/変更)

- [ ] 変更内容を明確化
- [ ] 影響範囲を評価
- [ ] リスクを評価
- [ ] 変更承認を取得
- [ ] バックアップを取得
- [ ] 変更を実施
- [ ] 変更後検証を実施
- [ ] 変更記録を作成
### **Step 6.6: 技術的負債管理（定期的）** (可変)

- [ ] 技術的負債をリストアップ
- [ ] 各負債の影響度を評価
- [ ] 優先度が設定されている
- [ ] 対応計画を作成
- [ ] 対応を実施（Phase 0-5のサイクルに組み込む）
- [ ] 対応結果を記録
