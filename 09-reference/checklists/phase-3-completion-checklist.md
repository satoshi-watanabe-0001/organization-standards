# Phase 3 完了チェックリスト

> コード実装、テスト、ドキュメント作成の完了確認

**目的**: Phase 3の実装が完了し、すべてのテストに合格し、コード品質基準を満たしていることを確認  
**使用タイミング**: Phase 4（レビュー・QA）への移行前  
**参照元**: [phase-3-implementation-guide.md](../../00-guides/phase-guides/phase-3-implementation-guide.md)

---

## ✅ 使い方

このチェックリストは、Phase 3（実装）のすべてのタスクが完了し、次のフェーズに移行する前に使用します。

### チェック方法
1. 各セクションを順番に確認
2. 完了している項目には `[x]` をマーク
3. すべての必須項目が完了したら、次のフェーズへ進行可能

### 未完了項目がある場合
- 該当項目を完了させる
- 技術的に困難な場合は、技術的負債として記録
- クリティカルな項目は必ず完了させる

## 4️⃣ データベースマイグレーション実装の確認

**データベース変更がある場合のみチェック**

### マイグレーションファイル作成

- [ ] マイグレーション命名規則に従った
- [ ] バージョン番号を適切に採番した
- [ ] 組織標準テンプレートを使用した

### SQLファイルのコメント（必須）

- [ ] ファイル冒頭に複数行コメントがある
- [ ] 【目的】セクションが記載されている
- [ ] 【ビジネス背景】にチケット番号が記載されている
- [ ] 【主な設計判断】が記載されている
- [ ] 【想定クエリパターン】が3つ以上記載されている
- [ ] 【インデックス方針】が記載されている
- [ ] 【セキュリティ考慮事項】が記載されている（該当する場合）

### テーブル定義

- [ ] SQLキーワードが大文字である
- [ ] 識別子が小文字スネークケースである
- [ ] 適切にインデントされている
- [ ] 全カラムにNOT NULL/DEFAULT等の制約が明示されている
- [ ] 外部キー制約にON DELETE/ON UPDATE動作が明示されている
- [ ] テーブルコメント（COMMENT ON TABLE）がある
- [ ] 関連チケット番号が記載されている
- [ ] 想定レコード数・成長率が記載されている

### カラムコメント

- [ ] 全カラムにCOMMENT ON COLUMNがある
- [ ] カラムコメントに形式・用途・制約が記載されている

### インデックス定義

- [ ] 各CREATE INDEXの前にインラインコメントがある
- [ ] インラインコメントに目的・想定クエリ・実行頻度・パフォーマンス効果が含まれる
- [ ] 各インデックスにCOMMENT ON INDEXがある
- [ ] COMMENT ON INDEXに簡潔な目的・想定クエリ・インデックスタイプ・推定効果が含まれる

### マイグレーション検証

- [ ] ローカル環境でマイグレーション実行を確認した
- [ ] ロールバック可能性を確認した（可能な場合）
- [ ] 想定クエリでパフォーマンステストを実施した
- [ ] インデックス効果をEXPLAIN ANALYZEで確認した

## 💡 チェックリストの使い方のヒント

### 1. 該当する項目のみチェック
- すべての項目が常に該当するわけではありません
- 「該当する場合のみチェック」と記載された項目は、該当しない場合はスキップ可能

### 2. 実装規模に応じた使用
- **小規模実装**（レベル3-4）: セクション1、7、8、9、14を重点的にチェック
- **中規模実装**（レベル1-2）: セクション1-3、7-10、14を重点的にチェック
- **大規模実装**（レベル0）: すべてのセクションを確認

### 3. 段階的なチェック
- 実装中に該当セクションを随時チェック
- Phase 3完了時に全体を最終確認

### 4. チームでの活用
- チェックリスト結果をチームで共有
- レビュー前の自己チェックに使用
- レビュアーもこのチェックリストを参照
