# Phase 5 完了チェックリスト

> 本番環境へのデプロイ完了確認

**目的**: Phase 5のデプロイが完了し、本番環境で正常に動作していることを確認  
**使用タイミング**: Phase 6（運用・保守）への移行前、またはPBI完了前  
**参照元**: [phase-5-deployment-guide.md](../../00-guides/phase-guides/phase-5-deployment-guide.md)

---

## ✅ 使い方

このチェックリストは、Phase 5（デプロイ）のすべてのタスクが完了し、次のフェーズに移行する前に使用します。

### チェック方法
1. 各項目を順番に確認
2. 完了している項目には `[x]` をマーク
3. すべての項目が完了したら、次のフェーズへ進行可能

### 未完了項目がある場合
- 該当項目の担当者に確認
- 必要に応じてタスクを追加・完了
- すべて完了するまで次フェーズへ進まない

## 🔄 Phase 5 のワークフロー: 4ステップ

### **Step 5.1: デプロイ前チェック** (15-20分)

- [ ] デプロイチェックリストの確認
- [ ] すべてのすべてのテストが合格している
- [ ] コードレビューが完了しているか
- [ ] デプロイ承認が得られているか
- [ ] データベースマイグレーションが準備できているか
- [ ] バックアップが取得されているか
- [ ] ロールバック手順が準備できているか
- [ ] デプロイ時間が適切か（メンテナンスウィンドウ等）
- [ ] 関係者に通知済みか
### **Step 5.2: デプロイ実行** (30-60分)

- [ ] デプロイスクリプトの実行
- [ ] デプロイスクリプトを実行
- [ ] デプロイ進捗をモニタリング
- [ ] エラーが発生していないか確認
- [ ] データベースマイグレーションが成功したか
- [ ] アプリケーションが起動したか
- [ ] 新バージョン（グリーン）を別環境にデプロイ
- [ ] トラフィックをブルー→グリーンに切り替え
- [ ] 問題があればブルーに戻す
- [ ] 新バージョンを一部のユーザーにのみデプロイ
- [ ] 問題がなければ段階的にロールアウト
- [ ] 問題があれば旧バージョンに戻す
- [ ] インスタンスを1つずつ新バージョンに更新
- [ ] 各インスタンス更新後にヘルスチェック
- [ ] すべてのインスタンスを順次更新
### **Step 5.3: デプロイ後検証** (20-30分)

- [ ] ヘルスチェックエンドポイントが正常に応答しているか
- [ ] 主要な機能が動作しているか（スモークテスト）
- [ ] エラーログが出力されていないか
- [ ] レスポンスタイムが正常範囲内か
- [ ] データベース接続が正常か
- [ ] 外部API連携が正常か
### **Step 5.4: リリースノート作成** (15-20分)

- [ ] リリースノートの作成と公開
- [ ] ステークホルダーへの通知
- [ ] 新機能をリストアップ
- [ ] バグ修正をリストアップ
- [ ] 既知の問題をリストアップ
- [ ] 破壊的変更がある場合は明記
- [ ] マイグレーション手順を記載（該当する場合）
- [ ] リリースノートを公開
- [ ] ステークホルダーに通知

## 🔍 実践的な活用方法

### 基本的なワークフロー

- [ ] **デプロイ前チェックを徹底**
- [ ] **デプロイを実行**
- [ ] **デプロイ後検証を実施**
- [ ] **リリースノートを作成**
