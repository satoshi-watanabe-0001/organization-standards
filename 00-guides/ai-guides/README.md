# ai-guides - AI向けプロンプト・ガイド

> AIエージェントの実行精度を高めるためのプロンプト、ガイドライン、チェックリスト集

**ディレクトリパス**: `/devin-organization-standards/00-guides/ai-guides/`

---

## 📖 概要

このディレクトリは、AI エージェント（Devin、Cursor、GitHub Copilot等）の**実行精度を高める**ためのプロンプト、ガイドライン、チェックリストを集約しています。

### 🎯 目的

1. **実行精度の向上**: AIの振る舞いを最適化するプロンプト
2. **一貫性の確保**: 全AIエージェントで統一されたガイドライン
3. **効率化**: 再利用可能なプロンプトテンプレート
4. **品質保証**: フェーズ別チェックリストによる品質担保

### 🌟 特徴

- **プロンプト中心**: AIの振る舞いを制御するプロンプト集
- **実践的**: 実際のタスクで即座に使えるテンプレート
- **段階的**: フェーズ別、タスク別に整理
- **進化型**: AI技術の進化に応じて月次で更新

**対象**: 🤖 AIエージェント、👨‍💻 人間のレビューアー

**構成**:
- **Phase 0**: 要件分析チェックリスト
- **Phase 1**: プロジェクト初期化チェックリスト
- **Phase 2**: 設計チェックリスト
- **Phase 3**: 実装チェックリスト
- **Phase 4**: レビュー・QAチェックリスト
- **Phase 5**: デプロイメントチェックリスト
- **Phase 6**: 運用・保守チェックリスト

#### いつ使う
- ✅ 各フェーズ完了前に確認
- ✅ 成果物の品質検証時
- ✅ レビュー時の確認項目として

#### 重要度
⭐⭐⭐⭐⭐ **最重要** - 品質保証の要

```markdown
# Phase 3 実装チェックリスト使用例

4. 不足項目をフィードバック
```

### 2. AI-CODING-INSTRUCTIONS.md (15KB)

**対象**: 🤖 AIエージェント

#### 📝 内容
コーディング時のAI向け詳細指示

**構成**:
- **一般原則**: DRY、KISS、SOLID
- **命名規則**: 変数、関数、クラスの命名
- **コメント**: いつ、どのようにコメントを書くか
- **エラーハンドリング**: 例外処理の方針
- **パフォーマンス**: 最適化のポイント
- **セキュリティ**: セキュアコーディングの注意点
- **言語別指示**: TypeScript、Python、Java等

#### いつ使う
- ✅ Phase 3（実装）開始前に読む
- ✅ コーディング中に参照
- ✅ コードレビュー指摘を受けた時に再確認

#### 重要度
⭐⭐⭐⭐⭐ **最重要** - コード品質の基盤

#### 使用例
```markdown
# TypeScript実装時の使用例

1. AI-CODING-INSTRUCTIONS.md を開く
2. 「TypeScript専用セクション」へ
3. 以下を確認:
   - 型定義の方針
   - interfaceとtypeの使い分け
   - Enumの使用ガイドライン
4. コーディング開始
5. 不明点があれば随時参照
```

### 7. AI-WORKFLOWS.md (18KB)

**対象**: 🤖 AIエージェント、👨‍💻 開発者

#### 📝 内容
ワークフロー詳細（AI実行フロー）

**構成**:
- **標準ワークフロー**: 通常の開発フロー
- **特殊ワークフロー**: Hotfix、リファクタリング等
- **並行処理**: 複数タスクの同時実行
- **エラーリカバリー**: エラー時の対応フロー
- **エスカレーション**: 人間への引き継ぎフロー
- **フェーズ遷移**: フェーズ間の移行判断

#### いつ使う
- ✅ 複雑なタスクを開始する時
- ✅ ワークフローに迷った時
- ✅ エラーが発生した時の対応を確認

#### 重要度
⭐⭐⭐⭐ 高 - ワークフロー理解に重要

#### 使用例
```markdown
# Hotfixワークフロー使用例

状況: 本番環境で緊急バグ発見

1. AI-WORKFLOWS.md を開く
2. 「特殊ワークフロー > Hotfix」セクション
3. フローを確認:
   Step 1: Phase 0 (最小限の分析)
   Step 2: Phase 3 (修正実装)
   Step 3: Phase 4 (最小限のテスト)
   Step 4: Phase 5 (緊急デプロイ)
4. フローに従って実行
5. 各ステップの完了基準を確認
```

**A**: AI-CODING-INSTRUCTIONS.mdとAI-CHECKLISTSは必須、他は推奨です。
