# 📅 フェーズ別ガイド

プロジェクトの各フェーズ（要件定義〜運用保守）における作業ガイドをまとめています。

## 📄 フェーズ一覧

### Phase 0: 要件定義・計画
**`phase-0-requirements-planning-guide.md`**

プロジェクトの要件定義と計画フェーズのガイドです。

**対象者**: プロジェクトマネージャー、ビジネスアナリスト

---

### Phase 1: プロジェクト初期化
**`phase-1-project-initialization-guide.md`**

プロジェクトの初期セットアップと環境構築のガイドです。

**対象者**: プロジェクトリーダー、テックリード

---

### Phase 2: 設計
**`phase-2-design-guide.md`**

システム設計全般のガイドです。

**対象者**: アーキテクト、設計担当エンジニア

#### Phase 2.1: 実装前設計
**`phase-2.1-pre-implementation-design-guide.md`**

実装開始前の詳細設計のガイドです。

#### Phase 2.2: 実装後設計
**`phase-2.2-post-implementation-design-guide.md`**

実装後の設計レビューと改善のガイドです。

---

### Phase 3: 実装
**`phase-3-implementation-guide.md`**

コーディングと実装フェーズの包括的ガイドです。

**対象者**: 開発エンジニア全員

---

### Phase 4: レビュー・QA
**`phase-4-review-qa-guide.md`**

コードレビューと品質保証のガイドです。

**対象者**: レビュアー、QA担当者、開発エンジニア

---

### Phase 5: デプロイメント
**`phase-5-deployment-guide.md`**

本番環境へのデプロイメントガイドです。

**対象者**: DevOpsエンジニア、デプロイ担当者

---

### Phase 6: 運用・保守
**`phase-6-operations-maintenance-guide.md`**

本番運用と保守フェーズのガイドです。

**対象者**: 運用担当者、保守エンジニア

---

## 📝 補助資料

### PHASE-CHECKLIST-TEMPLATE.md
各フェーズで使用できるチェックリストのテンプレートです。

**対象者**: 全フェーズのプロジェクトメンバー

---

## 🎯 推奨される使い方

1. **現在のフェーズに対応するガイドを読む**
2. **次のフェーズのガイドを予習する**
3. **PHASE-CHECKLIST-TEMPLATE.md** を使用して進捗管理
