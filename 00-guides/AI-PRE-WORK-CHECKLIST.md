# AI作業開始前チェックリスト

> AIエージェントが作業を開始する前に確認すべき事項の統合チェックリスト

**対象**: Devin、Cursor、その他の自律型AIエージェント  
**目的**: 作業開始前の確認漏れを防止し、効率的な開発を実現  
**使用タイミング**: 初回起動時、PBI受領時、各フェーズ開始時

---

## 📋 このチェックリストの使い方

### 3つのチェックリスト

このドキュメントには3種類のチェックリストがあります：

1. **初回起動時チェックリスト** - 最初の1回のみ実施（30-60分）
2. **PBI受領時チェックリスト** - PBIを受け取るたびに実施（5-10分）
3. **フェーズ開始時チェックリスト** - 各フェーズ開始時に実施（2-5分）

### 重要度の凡例

- 🔴 **必須** - スキップ不可、確認必須
- 🟡 **推奨** - 確認推奨、スキップ時は記録
- 🟢 **任意** - 必要に応じて確認

## 📋 PBI受領時チェックリスト（毎回実施）

**実施タイミング**: JIRA PBIを受け取った時  
**所要時間**: 5-10分  
**目的**: PBI理解と実行すべきフェーズの判断

### PBI基本情報の確認

- [ ] 🔴 **PBI情報を完全に読み込んだ**
  - [ ] タイトルを読んだ
  - [ ] 説明（Description）を読んだ
  - [ ] 受入基準（Acceptance Criteria）を読んだ
  - [ ] 技術的制約（Technical Constraints）を読んだ
  - [ ] 優先度（Priority）を確認した
  - [ ] ストーリーポイント（Story Points）を確認した
  - [ ] ラベル（Labels）を確認した
  - [ ] コンポーネント（Components）を確認した

- [ ] 🔴 **関連情報を確認した**
  - [ ] 関連Issue（Linked Issues）を確認した
  - [ ] 依存関係（Dependencies）を確認した
  - [ ] ブロッカー（Blockers）がないか確認した
  - [ ] 添付ファイル（Attachments）を確認した

### PBIタイプの判別

- [ ] 🔴 **PBIタイプを判別した**（以下のいずれか1つ）
  - [ ] タイプ1: 新規プロジェクト立ち上げ
  - [ ] タイプ2: 既存プロジェクトへの新機能追加
  - [ ] タイプ3: バグ修正
  - [ ] タイプ4: リファクタリング
  - [ ] タイプ5: 設計変更・アーキテクチャ改善
  - [ ] タイプ6: 緊急修正（Hotfix）
  - [ ] タイプ7: 実験的機能（Spike/POC）

### 実行すべきフェーズの判断

- [ ] 🔴 **フェーズスキップ判断フローを実施した**
  
  **Q1: 新規プロジェクトか？**
  - [ ] Yes → 全フェーズ実施 (Phase 0→1→2→3→4→5→6)
  - [ ] No → Q2へ

  **Q2: プロジェクト構造・技術スタック確定済みか？**
  - [ ] Yes → Phase 1 スキップ
  - [ ] No → Phase 1 実施

  **Q3: 設計変更が必要か？**
  - [ ] Yes → Phase 2 実施
  - [ ] No → Phase 2 スキップ

  **Q4: コード変更が必要か？**
  - [ ] Yes → Phase 3 実施
  - [ ] No → Phase 3 スキップ（設計のみの場合）

  **Q5: デプロイが必要か？**
  - [ ] Yes → Phase 4 → 5 実施
  - [ ] No → Phase 5 スキップ

- [ ] 🔴 **実行すべきフェーズを記録した**
  ```
  実行フェーズ: Phase [  ] → [  ] → [  ] → [  ]
  スキップフェーズ: Phase [  ], [  ] (理由: ________)
  ```

### 既存プロジェクト情報の確認（既存プロジェクトの場合）

- [ ] 🟡 **既存プロジェクト情報を確認した**
  - [ ] プロジェクトREADME.mdを読んだ
  - [ ] 既存アーキテクチャドキュメントを確認した

- [ ] 🟡 **既存コードベースを確認した**

  - [ ] 主要ファイル・ディレクトリを確認した
  - [ ] 依存関係（dependencies）を確認した

### 不明点・ブロッカーの特定

- [ ] 🔴 **不明点を特定した**

  - [ ] 🟡軽微な不明点: [      ]

## 💡 よくある質問（FAQ）

### Q1: 全てのチェックリストを毎回実施する必要がありますか？

**A**: いいえ。
- **初回起動チェックリスト** → 最初の1回のみ
- **PBI受領時チェックリスト** → PBIを受け取るたびに
- **フェーズ開始時チェックリスト** → 実施する各フェーズで

### Q2: チェックリストの項目をスキップできますか？

**A**: 重要度に応じて：
- 🔴 **必須** → スキップ不可
- 🟡 **推奨** → スキップ可能だが記録を推奨
- 🟢 **任意** → 必要に応じて実施

### Q3: 緊急修正（Hotfix）の場合はどうすればいいですか？

**A**: 簡易版で実施：
- PBI受領時チェックリスト → 5分で必須項目のみ
- Phase 0 → 簡易版（15-30分）
- Phase 3, 4 → 必須
- 後で詳細レビュー・ドキュメント補完を実施

### Q4: チェックリストの記録は必須ですか？

**A**: 推奨します：
- スキップしたフェーズの理由を明記
- 後から判断根拠を確認できる
- チーム内で一貫性を保つため
