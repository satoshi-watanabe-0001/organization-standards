# phase-guides - 開発フェーズ別ガイド

> Phase 0〜6の開発フェーズにおける実行手順とドキュメント参照ガイド

**ディレクトリパス**: `/devin-organization-standards/00-guides/phase-guides/`

---

## 📖 概要

このディレクトリは、ソフトウェア開発の**全7フェーズ**（Phase 0〜6）における詳細な実行手順とドキュメント参照ガイドを提供します。

### 🎯 目的

1. **明確なワークフロー**: 各フェーズで「何をすべきか」をステップバイステップで提示
2. **ドキュメント参照の最適化**: 各ステップで「どのドキュメントを参照すべきか」を明示
3. **品質保証**: 完了基準とチェックリストで品質を担保
4. **柔軟性**: フェーズスキップ可能、プロジェクト規模に応じた適用

### 🌟 特徴

- **軽量ナビゲーション型**: 各ガイド15-30KB、詳細は既存標準に委譲
- **実行型アプローチ**: ステップバイステップのワークフロー
- **優先度付き参照**: 🔴必須、🟡推奨、⚪参考の3段階
- **成果物明示**: 各フェーズの成果物を明確化

## 📋 Phase 別詳細

### Phase 0: 要件分析・企画

**ファイル**: [phase-0-requirements-planning-guide.md](./phase-0-requirements-planning-guide.md)  
**サイズ**: 15KB (459行)

#### 📝 概要
PBIから要件を分析し、開発の方向性を決定するフェーズです。

#### 🎯 主要タスク
1. **PBI分析**: PBIタイプ判定（Feature/Bug/Enhancement等）
2. **情報抽出**: 要件、制約、依存関係の特定
3. **不明点の特定**: エスカレーション判断
4. **依存関係の可視化**: 影響範囲の特定
5. **技術方針の決定**: 技術選定、アーキテクチャ方針
6. **リスク分析**: 技術的リスク、スケジュールリスク
7. **要件整理**: 要件分析書の作成（必要に応じて）

#### ⏱️ 推定時間
- **Feature**: 1-3時間
- **Bug**: 10-30分
- **Enhancement**: 30-90分

#### 🔗 主要参照ドキュメント
- 🔴 `../../10-governance/technology-radar.md` - 技術選定
- 🟡 `../../08-templates/requirements-analysis-template.md` - 要件分析書
- ⚪ `../../09-reference/glossary.md` - 技術用語集

#### ✅ 完了基準
- [ ] PBIタイプが特定されている
- [ ] 要件が明確に整理されている
- [ ] 不明点が解決またはエスカレーション済み
- [ ] 技術方針が決定されている
- [ ] リスクが特定され、対策が立案されている

#### 🔄 次フェーズへの移行
- **スキップ可能**: 小規模バグ修正（10-15分で完結）
- **必須**: 新機能開発、アーキテクチャ変更

## 🔄 フェーズ間の関係性

### 標準フロー（全フェーズ実行）

```
Phase 0 → Phase 1 → Phase 2 → Phase 3 → Phase 4 → Phase 5 → Phase 6
  ↓                                                              ↓
  ←──────────────────── インシデント発生時 ←─────────────────────
```

### スキップ可能なパターン

#### パターン1: 小規模バグ修正
```
Phase 0 (簡易版) → Phase 3 → Phase 4 → Phase 5
```
- Phase 1, 2をスキップ
- Phase 0も10-15分の簡易版

#### パターン2: 既存機能の小規模追加
```
Phase 0 → Phase 2 (簡易版) → Phase 3 → Phase 4 → Phase 5
```
- Phase 1をスキップ（既存プロジェクト）
- Phase 2も簡易版

#### パターン3: 緊急対応（Hotfix）
```
Phase 0 (最小限) → Phase 3 → Phase 4 (最小限) → Phase 5
```
- Phase 1, 2をスキップ
- Phase 4も最小限のテスト

### 🚀 クイックスタート

#### AIエージェントとして実行
```
1. AI-MASTER-WORKFLOW-GUIDE.md で現在のフェーズを確認
2. 該当するPhase Guideを開く
3. 「開始条件」を確認（条件を満たしているか）
```

## ❓ よくある質問

### Q1: すべてのフェーズを必ず実行する必要がありますか？
**A**: いいえ。上記の「フェーズスキップ判断基準」を参照してください。小規模なバグ修正などはPhase 1, 2をスキップ可能です。

### Q2: Phase Guidesと既存標準（01-10/）の違いは？
**A**: 
- **Phase Guides**: 「いつ」「どのドキュメントを」参照すべきかのナビゲーター
- **既存標準**: 「何を守るべきか」の詳細な標準とルール

### Q3: Phase 3（実装）が他より大きいのはなぜ？
**A**: 実装フェーズはタスク粒度別（Level 0-4）、タスクタイプ別、レイヤー別など多様なアプローチをカバーしているためです。

### Q4: 各Phaseの推定時間は目安ですか？
**A**: はい。プロジェクト規模、複雑度、チームのスキルレベルによって大きく変動します。初めての方は1.5-2倍の時間を見積もることを推奨します。

### Q5: Phase 6（運用・保守）はいつ実行しますか？
**A**: 本番環境へのデプロイ後、長期的に運用する場合に実行します。短期的な機能追加の場合はスキップ可能です。
