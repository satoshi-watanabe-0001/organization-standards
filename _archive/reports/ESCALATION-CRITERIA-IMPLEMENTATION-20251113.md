**配置先**: `/devin-organization-standards/00-guides/ESCALATION-CRITERIA-GUIDE.md`

**内容**:

#### 提案1: エスカレーション判断マトリクス
- **影響度（Impact Level）**: 4段階（Critical/High/Medium/Low）
  - 各レベルの定義と具体例を明記
- **緊急度（Urgency Level）**: 4段階（Immediate/Before Impl/Before Review/After Deploy）
  - 各レベルの判断基準を明記
- **判断マトリクス**: 4×4 = 16パターンの組み合わせで対応方針を決定
  - 🔴 即座に停止・エスカレーション
  - 🟠 実装前エスカレーション
  - 🟡 実装後レビュー要求
  - 🟢 記録して進行

#### 提案2: カテゴリ別の具体的閾値
**6カテゴリ × 3段階** の詳細基準:

1. **セキュリティ関連**
   - データ種別（個人情報、認証情報等）
   - 認証・認可（MFA、OAuth/SAML等）
   - 通信（TLS/SSL、暗号化等）

2. **アーキテクチャ・設計関連**
   - 影響範囲（3つ以上のサービス → Critical）
   - 技術選定（未承認技術 → Critical）
   - パターン変更（アーキテクチャ根本変更 → Critical）

3. **パフォーマンス関連**
   - レスポンスタイム（50%以上劣化 → Critical）
   - スケーラビリティ
   - リソース使用量（2倍以上増加 → Critical）

4. **ビジネスロジック関連**
   - データ整合性（金銭計算 → Critical）
   - ワークフロー（承認フロー変更 → Critical）
   - 計算ロジック（税金・ポイント計算 → Critical）

5. **組織標準遵守関連**
   - コーディング規約（未対応言語 → Critical）
   - ドキュメント（必須ドキュメント省略 → Critical）
   - プロセス遵守（Phaseスキップ → Critical）

6. **工数・スケジュール関連**
   - 工数増加（見積の2倍以上 → Critical）
   - 依存関係（ブロッカー発見 → Critical）

#### 提案3: Step 0.2 不明点処理フローの詳細化
現状の2ステップ → **4ステップの構造化プロセス**:

```
不明点を発見
  ↓
[ステップ1] 影響度レベルを判定 (Critical/High/Medium/Low)
  ↓
[ステップ2] 緊急度レベルを判定 (Immediate/Before Impl/Before Review/After Deploy)
  ↓
[ステップ3] マトリクスで対応方針を決定 (🔴/🟠/🟡/🟢)
  ↓
[ステップ4] 判断理由を記録（影響度・緊急度・対応方針・想定した標準対応）
```

**判断補助チェックリスト**:
- 影響度判定の質問（4つ）
- 緊急度判定の質問（3つ）

#### 提案4: 自己診断チェックリスト
**5セクション構成**:

- **セクションA（セキュリティ）**: 5問
  - 1つでも該当 → 🔴 Critical + Immediate
  
- **セクションB（アーキテクチャ）**: 6問
  - 2つ以上該当 → 🔴 即座にエスカレーション
  - 1つ該当 → 🟠 実装前エスカレーション

- **セクションC（ビジネスロジック）**: 6問
  - 2つ以上該当 → 🟠 実装前エスカレーション
  - 1つ該当 → 🟡 実装後レビュー要求

- **セクションD（組織標準）**: 5問
  - 1つでも該当 → 判定を1段階引き上げ

- **セクションE（UI・ログ）**: 5問
  - 3つ以上該当 & A/B/C/D項目0個 → 🟢 記録して進行

**判定フローチャート**: 5ステップの分岐判定

#### 提案5: ドキュメント間の整合性確保
- AI-MASTER-WORKFLOW-GUIDE.md をマスターソースとして定義
