# Phase開始前チェックリスト

> 各Phase開始前の共通確認事項

**目的**: 各Phaseを開始する前に、必要な準備が整っていることを確認  
**使用タイミング**: Phase 0-6のいずれかを開始する直前  
**参照元**: [AI-PRE-WORK-CHECKLIST.md](../../00-guides/AI-PRE-WORK-CHECKLIST.md)

---

## ✅ 使い方

このチェックリストは、**すべてのPhase**で共通して使用する開始前確認項目です。

### チェック方法
1. これから開始するPhaseに該当するセクションを確認
2. 各項目を順番にチェック
3. すべて完了したら、Phaseの作業を開始

### 未完了項目がある場合
- 前Phaseに戻って完了させる
- 必要なリソースを準備
- 準備が整うまで開始を保留

---

## 📋 全Phase共通の開始前確認

### 環境・ツール確認

- [ ] 開発環境にアクセスできる
- [ ] 必要なツールがインストールされている
- [ ] リポジトリにアクセス権限がある
- [ ] ドキュメント管理システムにアクセスできる
- [ ] JIRAにアクセスできる

### 前Phase成果物の確認

- [ ] 前Phaseの成果物がすべて完成している
- [ ] 前Phaseの完了チェックリストが完了している
- [ ] 前Phaseのレビューが承認されている
- [ ] 前Phaseで発生した課題が解決されている

### コミュニケーション確認

- [ ] チームメンバーと連絡が取れる
- [ ] ステークホルダーに状況報告できる
- [ ] 質問・相談先が明確である
- [ ] エスカレーション経路が分かっている

## 🎯 Phase別開始前確認

### Phase 0 開始前

- [ ] PBI受領時チェックリストが完了している
- [ ] PBIの必須情報がすべて揃っている
- [ ] 要件整理に必要な時間が確保できている

### Phase 1 開始前

- [ ] Phase 0の成果物（要件分析書、タスク分解リスト）が完成している
- [ ] リポジトリ作成権限がある
- [ ] 技術スタックが決定され、文書化されている（または候補が絞られている）

### Phase 2.1 開始前（事前実装設計）

- [ ] Phase 1の初期設定が完了している
- [ ] 設計に必要な技術情報が揃っている
- [ ] アーキテクチャ検討に必要な時間が確保できている（2日目安）

### Phase 2.2 開始前（事後実装設計）

- [ ] Phase 3の実装が完了している（後実施の場合）
- [ ] Phase 3の主要機能が実装済み（並行実施の場合）
- [ ] 実装中の設計変更が整理されている
- [ ] 設計書の最終化に必要な時間が確保できている

### Phase 3 開始前

- [ ] Phase 2.1の設計書が完成している
- [ ] 設計レビューが承認されている
- [ ] 実装環境が準備されている
- [ ] テスト環境が利用可能である
- [ ] CI/CDパイプラインが構築されている

### Phase 4 開始前

- [ ] Phase 3の実装が完了している
- [ ] すべてのすべてのテストが合格している
- [ ] コードがリポジトリにプッシュされている
- [ ] レビュアーが決定している

### Phase 5 開始前

- [ ] Phase 4のレビューが承認されている
- [ ] Phase 2.2の設計書最終化が完了している（該当する場合）
- [ ] デプロイ権限が付与されている
- [ ] 本番環境の状態が確認できている
- [ ] ロールバック手順が準備されている

### Phase 6 開始前

- [ ] Phase 5のデプロイが成功している
- [ ] 本番環境で正常動作が確認されている
- [ ] 監視ツールが設定されている
- [ ] 運用ドキュメントが準備されている

---

## 🔧 技術的準備確認

### ローカル環境

- [ ] 最新のコードをpullしている
- [ ] 依存ライブラリがインストールされている
- [ ] ローカルでビルドが通る
- [ ] ローカルでテストが実行できる

### リモート環境

- [ ] CI/CDパイプラインが正常に動作している
- [ ] 開発環境が利用可能である
- [ ] テスト環境が利用可能である
- [ ] 本番環境の状態が確認できている（Phase 5以降）
