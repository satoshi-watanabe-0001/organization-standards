# Phase 2.1 軽量版テンプレート

## 📋 このフォルダについて

このフォルダには、**Phase 2.1（基本設計・軽量版・実装前）** で使用する簡略化されたテンプレートが格納されています。

---

## 🎯 Phase 2.1 の目的

Phase 2.1は、**実装可否判断**と**技術的実現性の確認**を目的とした軽量設計フェーズです。

### Phase 2.1 と Phase 2.2 の違い

| 項目 | Phase 2.1（軽量版） | Phase 2.2（完全版） |
|------|---------------------|---------------------|
| **実施タイミング** | Phase 3（実装）の前 | Phase 3（実装）の後 |
| **目的** | 実装可否判断、技術的実現性確認 | 実装結果の完全な文書化 |
| **記載レベル** | 必要最小限の設計情報 | 詳細なパラメータ・設定値・実装詳細 |
| **作成時間** | 数時間〜1日 | 1〜2日 |
| **更新方針** | Phase 2.2で完全版に更新 | 最終版として保守 |

---

## 📂 テンプレート一覧

### 🔴 必須テンプレート（重要度A）

| No | テンプレート名 | 説明 | 記載レベル |
|----|----------------|------|------------|
| 1 | **システム構成図（軽量版）** | 主要コンポーネントの構成のみ | 詳細なパラメータは省略 |
| 2 | **アーキテクチャ設計書（軽量版）** | アーキテクチャパターンと主要技術のみ | ライブラリバージョンは省略 |

## 📝 軽量版テンプレートの使い方

### ステップ1: テンプレートの選択
Phase 2.1で必要な成果物のテンプレートを選択します。

### ステップ2: 必要最小限の情報のみ記載
以下の項目を**省略**してください：
- ✂️ 詳細なパラメータ・設定値
- ✂️ 具体的なライブラリバージョン
- ✂️ 詳細なインデックス・制約
- ✂️ 詳細な寸法・カラーコード・フォント
- ✂️ 全てのエラーコード詳細

以下の項目を**記載**してください：
- ✅ 主要コンポーネント・エンティティ
- ✅ アーキテクチャパターン
- ✅ 主要技術スタック
- ✅ 基本的なリレーション
- ✅ ワイヤーフレーム
- ✅ 主要エンドポイント

### ステップ3: 実装可否判断
軽量版設計書を基に、以下を判断します：
- 技術的に実現可能か？
- アーキテクチャ選定は適切か？
- データモデルは妥当か？
- UI/UXは実現可能か？

### ステップ4: Phase 3（実装）へ進む
実装可否が確認できたら、Phase 3（実装）に進みます。

### ステップ5: Phase 2.2で完全版に更新
Phase 3完了後、実装結果を反映して完全版に更新します（Phase 2.2）。

---

## 🚨 注意事項

### Phase 2.1での禁止事項
- ❌ 詳細設計レベルの情報を記載しない
- ❌ 実装詳細（具体的なコード構造等）を記載しない
- ❌ 完璧な設計書を目指さない

### Phase 2.1の目的を忘れない
Phase 2.1は**実装前の実現性確認**が目的です。完璧な設計書作成は**Phase 2.2**で行います。
